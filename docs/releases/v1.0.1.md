# Bonjou v1.0.1 Release Notes

## Highlights

- Adds robust path handling for `@file`, `@folder`, and `@multi`; quoted paths, `~`, and relative paths now resolve correctly.
- Tightens incoming transfer validation. Rejected payloads now emit explicit UI errors when peers are undiscovered or running mismatched builds.
- Displays the runtime version dynamically in the welcome banner.
- Updates README/HELP with clearer upgrade instructions and troubleshooting guidance for discovery and transfer issues.

## Upgrade Instructions

- **Homebrew**
  ```bash
  brew update
  brew upgrade bonjou
  ```
- **Scoop**
  ```powershell
  scoop update
  scoop update bonjou
  ```
- **Manual**: Download the platform binary from the release page, place it on the PATH, and rerun `bonjou`.

Ensure both peers upgrade to v1.0.1 or newer before exchanging messages or files.
